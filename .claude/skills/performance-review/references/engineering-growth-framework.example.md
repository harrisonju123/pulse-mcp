# Engineering Growth Framework (Example)

> **Note:** This is a generic example. Replace with your company's actual framework by creating `engineering-growth-framework.md` in this directory.

## Overview

An Engineering Growth Framework defines how software engineers at all levels are measured and evaluated. It provides clarity on expectations, competencies, and career progression paths.

## Core Competencies

### 1. Execution & Delivery
Getting the right things done the right way. Encompasses estimation, scoping, troubleshooting, incident management, and project execution.

**Levels:**
- **Novice**: Completes well-defined tasks with guidance
- **Intermediate**: Independently delivers features, handles ambiguity
- **Advanced**: Leads complex projects, de-risks proactively
- **Expert**: Drives org-wide delivery improvements

### 2. Skills & Knowledge
Technical excellence and continuous learning. Includes coding, architecture, debugging, security, performance, and domain expertise.

**Levels:**
- **Novice**: Learning fundamentals, needs code review guidance
- **Intermediate**: Writes production code, understands system design
- **Advanced**: Architects solutions, mentors on technical topics
- **Expert**: Sets technical direction, recognized expertise

### 3. Teamwork & Communication
Building relationships and collaborating effectively. Includes documentation, listening, empathy, and team building.

**Levels:**
- **Novice**: Communicates status, asks questions
- **Intermediate**: Documents work, collaborates across team
- **Advanced**: Facilitates discussions, resolves conflicts
- **Expert**: Shapes team culture, builds org-wide connections

### 4. Influence & Leadership
Multiplying impact through others. Includes mentorship, knowledge-sharing, consensus-building, and strategic thinking.

**Levels:**
- **Novice**: Shares learnings with team
- **Intermediate**: Mentors peers, influences team decisions
- **Advanced**: Drives cross-team initiatives, builds consensus
- **Expert**: Shapes org strategy, industry recognition

## Role Ladder

| Role | Typical Competency Expectations |
|------|--------------------------------|
| Junior Engineer | Novice/Beginner across competencies |
| Engineer | Beginner/Intermediate, growing independence |
| Senior Engineer | Intermediate/Advanced, leads projects |
| Staff Engineer | Advanced/Expert, cross-team impact |
| Principal Engineer | Expert across competencies, org-wide influence |

## Performance Ratings

Typical rating scale:
- **Needs Development**: Below expectations for level
- **Meets Expectations**: Performing at level
- **Exceeds Expectations**: Performing above level
- **Exceptional**: Consistently outstanding, ready for progression
