# Engineering Competency Rubric (Example)

> **Note:** This is a generic example. Replace with your company's actual rubric by creating `engineering-growth-rubric.md` in this directory.

## How to Use This Rubric

For each competency, find the level that best describes the engineer's consistent behavior. Look for patterns across multiple projects, not isolated instances.

---

## 1. Execution & Delivery

### Novice
- Completes well-defined tasks with clear requirements
- Asks for help when blocked
- Learns from code review feedback

### Beginner
- Breaks down tasks independently
- Estimates work with reasonable accuracy
- Identifies and escalates blockers early

### Intermediate
- Delivers features end-to-end with minimal guidance
- Handles ambiguous requirements effectively
- Proactively identifies and mitigates risks
- Participates in incident response

### Advanced
- Leads complex, multi-week projects
- Consistently delivers on time with high quality
- Coordinates across teams when needed
- Drives post-incident improvements

### Expert
- Drives delivery improvements across the organization
- Trusted to lead highest-priority initiatives
- Defines best practices for project execution
- Mentors others on effective delivery

---

## 2. Skills & Knowledge

### Novice
- Learning language fundamentals and tooling
- Writes code that works with guidance
- Understands basic debugging techniques

### Beginner
- Writes clean, tested code
- Understands codebase patterns
- Debugs issues independently
- Familiar with system architecture

### Intermediate
- Designs and implements features end-to-end
- Makes sound technical tradeoffs
- Reviews code effectively
- Considers security and performance

### Advanced
- Architects complex systems
- Deep expertise in key areas
- Sets technical standards for team
- Debugs the most challenging issues

### Expert
- Recognized technical authority
- Defines architectural direction
- Innovates solutions to hard problems
- Influences industry practices

---

## 3. Teamwork & Communication

### Novice
- Communicates status in standups
- Asks questions when stuck
- Responsive to feedback

### Beginner
- Documents work clearly
- Collaborates effectively with teammates
- Gives constructive code review feedback

### Intermediate
- Facilitates team discussions
- Writes documentation others rely on
- Builds relationships across teams
- Helps onboard new team members

### Advanced
- Resolves team conflicts constructively
- Drives cross-team collaboration
- Creates team processes that stick
- Trusted advisor to many colleagues

### Expert
- Shapes engineering culture
- Builds bridges across organizations
- Models excellent communication
- Elevates documentation standards org-wide

---

## 4. Influence & Leadership

### Novice
- Shares learnings with teammates
- Open to mentorship

### Beginner
- Helps teammates when asked
- Contributes ideas in discussions
- Takes ownership of commitments

### Intermediate
- Mentors junior engineers
- Influences team technical decisions
- Drives improvements to team practices
- Speaks up about concerns

### Advanced
- Leads cross-team initiatives
- Builds consensus on difficult decisions
- Mentors engineers across teams
- Recognized leader in their area

### Expert
- Shapes organizational strategy
- Multiplies impact of entire teams
- Industry recognition (talks, writing, OSS)
- Develops other leaders

---

## Using the Rubric for Reviews

1. **Gather evidence**: Collect specific examples for each competency
2. **Find the level**: Match consistent behavior to rubric descriptions
3. **Note gaps**: Identify areas where behavior varies from level
4. **Set goals**: Use next level as development targets
